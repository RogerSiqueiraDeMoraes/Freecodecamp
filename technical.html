PHP Tags

When PHP parses a file, it looks for opening and closing tags, which are <?php and ?>, telling PHP to start and stop interpreting the code. Parsing in this way allows PHP to be embedded in various documents, as everything outside these tags is ignored by the parser.

PHP includes a short echo tag <?=, which is shorthand for <?php echo.

Example #1 PHP Opening and Closing Tags


<?php echo 'Use these tags to serve PHP code in XHTML or XML documents'; ?>
You can use the short echo tag to <?= 'print this string' ?>.
It's equivalent to <?php echo 'print this string' ?>
<? echo 'Short tags work if short_open_tag is enabled'; ?>

Short tags are enabled by default but can be disabled via the `short_open_tag` directive in php.ini. For compatibility, it's recommended to use the standard tags (<?php ?> and <?= ?>).

Note:

If a file contains only PHP code, it's better to omit the closing PHP tag at the end of the file to prevent accidental whitespace or new lines that could cause unwanted effects.

Escaping from HTML

Everything outside PHP tags is ignored by the parser, allowing PHP to be embedded in HTML documents to create templates. For example: 


<p>This is ignored by PHP.</p>
<?php echo 'This is parsed.'; ?>
<p>This is also ignored.</p>

Using structures with conditions allows for more complex output handling:

Example #2 Escaping with Conditions


<?php if ($expression == true): ?>
This shows if true.
<?php else: ?>
This shows otherwise.
<?php endif; ?>

Note:

For XML or XHTML, always use the standard PHP tags (<?php ?>).

Instruction Separation

PHP statements must end with a semicolon (;). The closing PHP tag automatically includes a semicolon for the last line of a PHP block.

Example #3 Instruction Separation


<?php echo "Text with no newline"; ?>
No newline
<?= "Now with newline" ?>

Output:


Text with no newlineNo newline
Now with newline

Note:

Omitting the closing PHP tag is helpful to prevent unwanted whitespace and ensure better integration when including or requiring files.

Comments

PHP supports 'C', 'C++', and Unix shell-style comments. Examples: 


<?php
// C++ style comment
/* Multi-line comment */
# Unix shell-style comment
?>

Avoid nesting 'C'-style comments as it causes errors: 


<?php
/*
   echo 'Test'; /* Nested comment causing an error */
*/
?>
